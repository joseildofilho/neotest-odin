# neotest-odin

Neotest adapter for the Odin programming language.

`neotest-odin` integrates Odin's test runner with the Neovim testing ecosystem, allowing you to discover, run, and debug Odin tests directly from Neovim using Neotest.

## Features

- Automatic discovery of Odin test files (`*_test.odin`)
- Test discovery using Tree-sitter
- Run individual tests
- Run all tests in a package
- Integration with Neotest UI
- DAP debugging support
- JSON test report parsing

## Requirements

- Neovim >= 0.10
- Odin compiler
- Tree-sitter Odin parser
- neotest
- nvim-nio
- nvim-dap (optional, for debugging)

## Installation

Using `lazy.nvim`:

```lua
{
    "nvim-neotest/neotest",
    dependencies = {
        "nvim-neotest/nvim-nio",
        "joseildofilho/neotest-odin",
    },
    config = function()
        require("neotest").setup({
            adapters = {
                require("neotest-odin"),
            },
        })
    end,
}
```

## Tree-sitter

This adapter requires the Odin Tree-sitter parser.

Install it with:

```vim
:TSInstall odin
```

## Usage

### Run nearest test

```lua
require("neotest").run.run()
```

### Run current file

```lua
require("neotest").run.run(vim.fn.expand("%"))
```

### Run current package

```lua
require("neotest").run.run(vim.fn.getcwd())
```

### Debug nearest test

```lua
require("neotest").run.run({
    strategy = "dap"
})
```

### Toggle test summary

```lua
require("neotest").summary.toggle()
```

### Toggle output panel

```lua
require("neotest").output_panel.toggle()
```

## Supported Test Format

The adapter discovers tests declared with the `@test` attribute.

Example:

```odin
package math

import "core:testing"

@test
add_numbers :: proc(t: ^testing.T) {
    testing.expect(t, 1 + 1 == 2)
}
```

Files must follow the naming convention:

```text
something_test.odin
```

## How It Works

The adapter:

1. Detects Odin test files.
2. Uses Tree-sitter to discover test procedures.
3. Executes tests through Odin's test runner.
4. Reads JSON reports generated by Odin.
5. Converts results into Neotest-compatible output.

## Debugging

Debugging is supported through Neotest's DAP integration.

The adapter builds a temporary test binary and launches it using your configured DAP adapter.

A typical setup uses CodeLLDB:

```lua
require("neotest").run.run({
    strategy = "dap"
})
```
